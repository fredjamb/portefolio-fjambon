PROJET GEDAP

Dans le cadre des 3 stages suivis au cours de la formation au GRETA, au sein d'une équipe composée par 4 stagiaires, notre mission fut de créer un logiciel permettant la gestion des apprentis au sein du GRETA-CFA QUIMPER OCCIDENTALE. Celui-ci permettant l'inscription des apprentis dans une base de données ainsi que toutes les opérations inhérentes au suivi du parcours professionnel.

La Structure informatique

Le projet est hébergé en local sous Laragon, dans le répertoire www, dans un sous-répertoire GEDAP. (sur une installation standard, votre chemin d’accès au projet local est : C:\laragon\www\GEDAP)

Dans le répertoire GEDAP, on trouve :

  1. un dossier « spdf » qui contient l’API spdf permettant de générer les pdf à partir du code php. Il ne doit rien contenir d’autre 
  2. un dossier « .git » caché qui contient des fichiers nécessaires au fonctionnement du projet avec github 
  3. des fichiers de configuration de git/github (readme.md, .gitignore, .env.development)
  4. un dossier « vues » qui contient les fichiers de ressources (php, ...) en affichage dédiés à la consultation et à la connexion
  5. un dossier « traitement » qui contient les fichiers de ressources (php, …) nécessaires à la création et à la modification et à la gestion des pdf 
  6. un dossier « controleurs » qui contient les fichiers php qui effectuent des actions invisibles à l’utilisateur 
  7. un dossier « images » qui contient les fichiers graphiques 
  8. un dossier « css » qui contient les fichiers css.
La sécurité

- le site est protégé par mot-de-passe et des droits sont attribués aux utilisateurs par un administrateur

- la stratégie de mot-de-passe établit les standards suivants :

  1. au minimum 12 caractères alphanumérique, incluant les caractères accentués UTF8 et les symboles (&»[@, etc.)
  2. le mot-de-passe doit être changé tous les X mois, un rappel est affiché à l’utilisateur après l’identification si la date est dépassée (et/ou l’utilisateur est envoyé sur la page de compte)
  3. le mot-de-passe ne peut être identique au mot-de-passe précédent (au moins X caractères de différence)

- l’accès à la base de données est protégée par un fichier de configuration (config.php) située dans le répertoire racine du projet (GEDAP).

Connexion

L’interface de connexion comprend une page php et un fichier contrôleur. L’utilisateur accède au progiciel par la page connexion.php. La saisie du login/pass détermine ses droits d’accès.

connexion.php
  1. formulaire : login, mot-de-passe, bouton submit, bouton reset
  2. un lien : « mot-de-passe oublié ? » (action à déterminer)
  3. envoi vers le contrôleur test_cnx.php

test_cnx.php
  1. accès à la base de données ‘gedap’, table ‘user’
  2. test les variables POST envoyées par connexion.php en fonction du résultat (droits : A, C, M), envoie automatiquement vers l’interface d’administration (menu.php), l’interface de consultation (filtre.php), l’interface de modification (menu.php)
Administration

L’interface d’administration est composée de quatre pages :

  1. une page menu.php,
  2. une page de création d’année n_annee.php,
  3. une page de création de formation n_formation.php,
  4. une page de gestion de compte compte.php.

Toutes les pages d’administration sont dotées :

  1. d’un bouton déconnexion, situé dans la partie droite du bandeau « logos »
  2. d’un bandeau de cheminement, sous le bandeau « logos », indiquant la page en cours et permettant de revenir sur la/les page(s) précédente(s)

La page: Menu

  1. teste les droits de l’utilisateur et affiche des éléments de menu différenciés
  2. accès à la base de données ‘gedap’, table ‘user’
  3. un lien ‘nouvelle année’ envoie vers la page n_annee.php
  4. un lien ‘nouvelle formation’ envoie vers la page n_formation.php
  5. un formulaire permet de créer un nouveau compte : input login, mot-de-passe, droits
